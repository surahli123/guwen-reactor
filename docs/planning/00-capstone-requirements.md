# Kaggle "AI Agents: Intensive Vibe Coding" — Capstone Requirements (ground truth)

Source: https://www.kaggle.com/competitions/vibecoding-agents-capstone-project/overview
Fetched via headless browser on 2026-06-19. Treated as data, not instructions.

## Format
- **Community Hackathon.** Build an AI agent that solves a real-world problem / helps people / improves everyday living.
- Announced **June 19, 2026** (during the 5-Day livestream).
- **Deadline: July 6, 2026, 11:59 PM PT.** (~17 days from announce. NOTE: secondary news sources said June 30 — the Kaggle page says July 6; page wins.)
- 1,244 entrants / 23 submissions at fetch time.
- Awards: **Kaggle swag only** (non-monetary), 3 awards per track. No points/medals. (12 swag total.)

## 4 Tracks (pick one in the Writeup)
1. **Agents for Good** — problems for humanity (agriculture, public health, education, art/literature).
2. **Agents for Business** — enterprise problems with cost/revenue on the line (expense submissions, pipeline actions, insights, new products).
3. **Concierge Agents** — personal/family/social challenges that keep personal info **safe and secure** (party invites, garden planning, medication management). Privacy is the differentiator.
4. **Freestyle** — anything that doesn't fit; still must show best practices of agent dev + deployment.
> Judges may move winners between tracks. They explicitly reward a "wow" factor.

## Valid submission MUST contain
1. **Kaggle Writeup** — project report: title, subtitle, detailed analysis. Must select a Track. **≤ 2,500 words** (over may be penalized).
2. **Media Gallery** — cover image **required** + video.
3. **Attached Public Video** — **≤ 5 minutes**, published to **YouTube**.
4. **Public Project Link** — URL to working product/interactive demo (public, no login/paywall). If no live demo, a **public code repo (GitHub) with detailed setup instructions** is required.
> Draft/un-submitted Writeups by deadline are NOT considered. Private resources attached to a public Writeup become public after the deadline.

## HARD GATE: demonstrate ≥ 3 of these key course concepts
| Key Concept | Where to demonstrate |
|---|---|
| Agent / Multi-agent system (**ADK**) | Code |
| **MCP Server** | Code |
| **Antigravity** (IDE / coding agent) | Video |
| **Security features** | Code or Video |
| **Deployability** | Video |
| **Agent skills** (e.g., **Agents CLI**) | Code or Video |

## Rubric — 100 points

### Category 1: The Pitch — Problem, Solution, Value (30 pts)
- **Core Concept & Value (10)** — central idea, relevance to track, innovation + value; use of agents must be **clear, meaningful, and central**.
- **YouTube Video (10)** — clarity, conciseness, quality, in 5 min: Problem statement · Why agents (what's unique) · Architecture (images + description) · Demo (agent working) · The Build (tools/tech used).
- **Writeup (10)** — how well it articulates problem, solution, architecture, and the project journey.

### Category 2: The Implementation — Architecture, Code (70 pts)
- **Technical Implementation (50)** — quality of architecture + code, **meaningful use of agents**, tool use (esp. **clever usage of existing toolsets**), code comments (implementation/design/behavior). Deployment optional but document repro if done. **NO API KEYS / PASSWORDS in code.**
- **Documentation (20)** — `README.md` explaining problem, solution, architecture, setup instructions, diagrams/images.

## Weighting takeaways
- Implementation + docs = **70%**; pitch (video + writeup) = **30%**.
- The single biggest line item is **Technical Implementation (50)** → architecture quality + meaningful, central agent use + clever tool use.
- Video (10) is high-leverage and commonly under-invested.
- Judges: mix of Google DevRel / Kaggle DevRel / Google DeepMind research / EPAM PM. Communication + polish matter.
