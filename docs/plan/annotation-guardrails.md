# Phase-1 标注护栏 (Annotation Guardrails) — gate 校准前提

**适用范围**：3 个被测场景 `管宁割席 / 咏雪 / 道旁苦李`。
**来由**：这些护栏由本次研究的**对抗审查（adversarial labelability audit）**逐则挖出。
写 `canon_gold.yaml` 时逐场景套用；它们同时是 gate 的 **calibration 锚点**。
**配套**：双标注流程见 `gold-cross-validation-protocol.md`。

---

## A. 通用结构规则 (所有场景通用)
1. **speech-act ≠ denotation**：人物说的话/判断，字面事实只是「X 说了 Y」；Y 本身是否为真**不是**字面（除非原文另行坐实）。
2. **动机/评价不进字面**：只标**动作**与**言语**。动机、褒贬、性格判断 → advisory interpretation 层，不进 hard-gate 的 `atomic_fact`。
3. **本名/消歧进推论层**：原文只有官称/小字/亲属称谓时，补出的本名 = `inferred`，不当 denotation。
4. **动作+未遂绑成一条**：「戳/没戳中」「踩/没踩中」算**一条** structured-claim，别拆两条 → 防重叠计分（精确率分母被稀释）。
5. **主语锁位**：省略主语的指代（如「乃至」「取之」）→ 锚到原文 span + 保留主语开放，不当「原文明说」。
6. **切分边界统一**：以「说话人转换 / 离散动作」为**唯一**节拍边界；A、B 两位标注者用**同一**边界，否则一致率假性下降。

## B. 每场景专属坑

### 咏雪 / 咏絮（clean）
- **禁**「柳絮优于撒盐」「谢安更赏侄女」当事实——原文只『公大笑乐』，孰优孰劣是千年文评公案。
- 本名 谢安 / 谢朗(胡儿) / 谢道韫 → 推论层。
- 篇末「即公大兄无奕女，左将军王凝之妻也」= 旁白身份补注 → **背景事实**层，非情节节拍。

### 道旁苦李（caution — 推论坑最集中）
- 「此必苦李」= 王戎的 **speech-act / 主张**，**不可**标成「李确实是苦的」。
- 「信然」只标「果如其言」，**不可**外推「因为长在路边所以苦」的因果。
- 两次「取之」主语不同（诸儿竞走取 vs 取以验证）→ 锁位置；第二次主语 underspecified，**保持开放**（最不可能是「不动」的王戎）。
- 「折枝」= 压弯枝条（非折断）；「不动」= 没去摘（把可接受表述边界写清：「站着不动」「没去摘」都算对）。
- **禁**「王戎聪明 / 有雅量」评价当事实（编者归「雅量」是 connotation）。

### 管宁割席（核心被测）
- 「华歆贪慕 / 慕荣」类动机评价**不进**字面——原文只有『捉而掷去之』(动作)、『废书出看』(动作)。
- 取证的 10 条 denotation 全是动作与言语，**零动机、零评价** → final gold 必须长这样。
- gold 走路径 2+3 独立核对；第二位读者交叉验证随后补，暂揭露 `no-IAA`。

## C. 落地：护栏 ↔ gate 的双向证明
这些护栏不只是给标注者看，它们是 gate 校准的两端锚点：
- **Specificity test（Contract G）**：一条**合规的** hedged interpretation（如「文中暗示华歆为荣华所动 *(hedged)*」，cite 支持事实）在干净 run 上 **不应被 block** → 证明 gate 没过度拦。
- **Drift suite（4b）**：一条**违反护栏的**伪造 claim（如把「此必苦李」写成既成事实、或给管宁割席补上「华歆贪财」）**应被 block** → 证明 gate 拦得住。

→ 护栏定义了「什么是污染」；specificity + drift 两套测试证明 gate 正好卡在这条线上，不松不紧。
