## 二、合并 findings(按严重度去重;标注共识强度)

### BLOCKER(不改不能动工)

**B1. 地图节点错配:驿站=beat,应=文章。**(3/3 线全中,最强共识)

**B2. 信任状态越界:灯火全亮 + 印章无口径 = 多篇后即谎。**(信任UX + Codex 教师评论家 + baoyu 同中)
节点=文章后,G01/G02/G03 同样亮灯=视觉断言"三篇已核实",而 G02/G03 读本未创作、gold 待人工复核(canon_gold.yaml:56)。改法:**灯火=按篇核实状态**(亮琥珀=据链完整+REVIEWED,现仅 G01;暗/余烬=读本待创作);**印章按篇作用域** + 携带 v1 的口径免责句(index.html:424-427「徽章报告的是引擎对结构化事实的判定;读本是忠实改写」);G02/G03 显示「读本待创作·考据链未建立」而非 0/9。

**B3. 跨篇动线为零。**(交互 BLOCKER + Codex IA)
两原型没有任何「篇目列表/回到首页/下一篇」机制。改法:地图本身即 hub(默认落地页),另加**风华榜式索引面板**(右上控件签牌,列全篇:篇名/出处/beat 数/已核实数)作寻址+a11y 兜底——**中性目录排序(門類/篇名),严禁做成忠实度排行**(违徽章口径)。

**B4. 标题书法字跨机静默垮。**(baoyu 升为 BLOCKER;美学维度 MAJOR)
`--f-title` 全 macOS-only,Win/Linux 评委(=真实观众)看到的竖排行楷退化为普通衬线,山河卷第一信号塌掉。改法:`pyftsubset` 裁 **LXGW 文楷子集**(覆盖全部篇名+chrome 用字)base64 内嵌,几 KB。

**B5. 内容诚信:不得伪造地理。**(baoyu 红旗 + IA + Codex hard question #1)
参考站的城池位置是真地理;世說新語轶事无地理坐标,照搬中国地图把「割席」钉在某座真实城池=编造,自相矛盾于「言必有據」。改法见"三、待拍板 Q1"。

## 三、保留的分歧 → 需要 owner 拍板的 3 个问题(附推荐)

**Q1. 地图隐喻的语义轴是什么?**(Codex hard question #1 + baoyu 红旗)
- 方案 a(IA 维度推荐):**按《世說新語》門類分区**——数据现成(source_id:G01=德行、G02=言語、G03=雅量),每門一片山水/城群,可扩到 36 門上百则,节点位置有真语义。
- 方案 b(baoyu):构图摆位 + 小字声明「布局为叙事,非地理」。
- **推荐:a+b 合并**——門類分区给语义轴,再加一行非地理声明,双保险且完全"有据"(門類本来就是原书的真实分类)。

**Q2. 开场展卷动画留不留?**(策展人 vs 教师/信任UX 的真分歧)
- 策展人:仪式感立世界观;信任UX:零证据载荷,把 time-to-first-trust 往后推;教师:浪费评委时间。
- **推荐:留但降级**——首访播一次、可跳过、≤3.4s;localStorage 记忆;之后一律 ≤0.8s;录 demo 视频时手动重播(它在视频里就是 5 秒的开场镜头,价值最大化的场合恰是视频不是网页)。

**Q3. 10+ 篇时的寻路承诺?**(Codex hard question #3)
- **推荐:門類空间分区(Q1a)+ 风华榜文字索引(中性排序)** 双轨;地图浅缩放即可(栅格真迹深缩放与单文件体积冲突——工程剧评人+IA 一致警告)。3 篇时地图必然稀疏:靠真迹丰富底+参考本就 60-80% 留白托底,**不补假签牌**。
- 附注(体积预算,动工前定):单张 1920px 级 JPEG crop base64 后约 400-800KB,单文件总预算建议 ≤3MB——足够 C 的固定舞台,不够 B 的长卷,这也是 verdict 的工程依据之一。
